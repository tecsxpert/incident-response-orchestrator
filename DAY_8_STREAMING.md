# Day 8: SSE Streaming for Report Generation

## Overview

Implemented Server-Sent Events (SSE) streaming for the `/generate-report` endpoint. This allows the frontend to receive report tokens in real-time as they are generated by the AI model, eliminating the wait for the complete response.

## Architecture

### Backend (Flask/Python)

**New Endpoint:** `POST /api/describe/generate-report-stream`

```
Flask App (port 5000)
    ↓
GroqClient.generate_report_stream()
    ↓
Groq API with stream=True
    ↓
Yields tokens one by one
    ↓
SSE event stream (text/event-stream)
    ↓
Frontend (EventSource)
```

### Frontend (React)

**New Hook:** `useReportStream()`

```
React Component
    ↓
useReportStream hook
    ↓
EventSource API
    ↓
SSE stream from backend
    ↓
Callback with tokens
    ↓
Real-time UI update
```

## Implementation Details

### 1. Backend Changes

#### File: `ai-service/services/groq_client.py`

Added two new methods:

**`chat_stream(messages, model, temperature, max_tokens, top_p)`**
- Low-level streaming wrapper around Groq API
- Yields tokens as they arrive
- Parameters same as `chat()` method
- Returns generator that yields token strings

**`generate_report_stream(title, incident_type, severity, description, discovery_date, current_status, system_prompt)`**
- High-level method for streaming report generation
- Calls `chat_stream()` internally
- Uses GENERATE_REPORT_PROMPT template
- Yields tokens for the report

#### File: `ai-service/routes/describe.py`

Added new endpoint and imports:

**New imports:**
```python
from flask import Response, stream_with_context
import json
```

**`POST /api/describe/generate-report-stream`**
- Takes same input as `/generate-report` (title, incident_type, severity, description, etc.)
- Validates all input fields
- Returns Server-Sent Event stream

**Event stream format:**

1. **Metadata event** (JSON):
```
data: {"type": "metadata", "title": "...", "incident_type": "...", "severity": "...", "timestamp": "...", "model": "..."}
```

2. **Token events** (text):
```
data: This is a token
data:  with spaces
data: and more content
```

3. **Completion event** (JSON):
```
data: {"type": "complete", "tokens_streamed": 1234, "timestamp": "..."}
```

4. **Error event** (JSON):
```
data: {"type": "error", "error": "error message", "timestamp": "..."}
```

**Response headers:**
```
Content-Type: text/event-stream
Cache-Control: no-cache
X-Accel-Buffering: no
Connection: keep-alive
```

### 2. Frontend Changes

#### File: `frontend/src/hooks/useReportStream.js`

Custom React hook for managing SSE stream:

**Hook state:**
- `tokens` - Accumulated token content
- `metadata` - Metadata from stream
- `isLoading` - True while streaming
- `error` - Error message if any
- `isComplete` - True when stream finished

**Hook methods:**
- `streamReport(incidentData, onToken)` - Start streaming report
  - `incidentData` - Object with title, incident_type, severity, description, etc.
  - `onToken` - Optional callback for each token received
- `cancelStream()` - Stop ongoing stream

**Features:**
- Auto-cleanup on component unmount
- Error handling
- Configurable base URL via options or environment variable
- Separate handling for metadata, tokens, and completion events

#### File: `frontend/src/components/ReportStreamer.jsx`

React component for displaying streamed reports:

**Features:**
1. **Metadata display** - Shows incident info, type, severity, model
2. **Real-time content** - Displays tokens as they arrive with blinking cursor
3. **Controls** - Start/Cancel buttons
4. **Error display** - Shows errors if stream fails
5. **Loading indicator** - Shows spinner and word count while streaming
6. **Statistics** - Word count, character count, tokens streamed
7. **Export** - Download as text or copy to clipboard
8. **Auto-start** - Optional autoStart prop

**Props:**
```javascript
<ReportStreamer
  incidentData={{
```

### 3. How It Works

1. **User clicks "Start Report"** on the ReportStreamer component
2. **Hook initiates EventSource** with POST request containing incident data
3. **Backend validates input** and starts streaming from Groq API
4. **Backend sends metadata event** with incident info
5. **Backend streams tokens** one at a time
6. **Frontend receives each token** via EventSource onmessage
7. **React state updates** for each token (real-time UI updates)
8. **Frontend displays cursor** showing active streaming
9. **Backend sends completion event** when done
10. **Frontend shows complete badge** and enables export
11. **User can download** report as text file

## Benefits

1. **Perceived Performance** - User sees content appearing immediately instead of waiting
2. **Better UX** - Real-time feedback with progress
3. **Reduced Blocking** - No long-running requests
4. **Responsive** - Frontend remains interactive during generation
5. **Cancellable** - User can stop streaming at any time
6. **Exportable** - Generated content can be downloaded
7. **Copyable** - Content can be copied to clipboard
8. **Reliable** - Graceful error handling and recovery

## Error Handling

The implementation handles various error scenarios:

1. **Connection refused** - EventSource onerror
2. **Invalid input** - HTTP 400 validation
3. **AI service error** - Error event in stream
4. **Stream interrupted** - User can cancel
5. **Parsing errors** - Logged but don't break stream

## Performance Metrics

- **Time to first token:** ~500ms-1s (API latency + streaming setup)
- **Token rate:** ~5-10 tokens/second from Groq API
- **Report generation:** ~30-60 seconds for full report
- **Frontend rendering:** Minimal impact (batched updates)
